# 一、项目定位与背景

## 1.1 项目定位

**产品名称**：蓝心校园 AI 管家

**产品形态**：vivo 快应用（免安装、即点即用）

**核心定位**：面向大学生的**课堂多模态智能笔记 + 朋友式时间管理** AI 管家

**品牌人设**：既是帮你搞定学习的「智能管家」，又是懂你、会鼓励你的「校园朋友」

**一句话 slogan**：*你的校园学习搭子，更懂你的 AI 管家*

## 1.2 产品愿景

让每个大学生拥有一个 24 小时在线的 AI 学习伴侣——课堂自动记笔记、DDL 智能提醒、期末自动生成复习资料，把学生从繁琐的信息整理中解放出来，聚焦真正的学习本身。

## 1.3 背景分析

### 政策背景
- 教育部《新一代人工智能发展规划》推动 AI+教育深度融合
- 高校数字化转型加速，智慧课堂基础设施普及
- AIGC 技术成熟，端侧大模型部署成为行业趋势

### 技术背景
- 蓝心大模型（Blue LM）云端 API + 3B 端侧模型 SDK 双模架构
- vivo 快应用生态 7 亿+ 终端覆盖，零安装即用
- 多模态大模型（视觉、语音、文本）能力成熟
- 端侧推理芯片（骁龙 8 Gen、天玑 9000+）支撑 3B 模型流畅运行

### 校园痛点背景
- 大学生日均 4-6 节课，笔记整理耗时占学习总时间 35%
- 67% 大学生存在 DDL 拖延症（2024 中国大学生学习行为报告）
- 期末复习资料零散，缺乏系统性整理
- 市面上缺乏真正 AI-native 的校园学习工具

# 二、目标用户与痛点分析

## 2.1 目标用户画像

| 维度 | 描述 |
|------|------|
| **年龄** | 18-24 岁，大一至大四本科生 |
| **设备** | vivo / iQOO 手机用户（快应用原生优势） |
| **学习场景** | 课堂听讲、自习复习、期末冲刺、网课学习 |
| **技术敏感度** | 高，日常使用 AI 工具（ChatGPT、文心一言、豆包） |
| **核心诉求** | 提效、减负、不挂科、拿高分 |
| **情感诉求** | 需要陪伴感、鼓励式学习、避免孤独感 |

## 2.2 痛点矩阵

| 痛点 | 场景描述 | 频次 | 强度 | 现有方案 |
|------|---------|------|------|---------|
| **课堂笔记跟不上** | 老师 PPT 翻页快，手写/打字来不及，课后笔记残缺 | 每天 | ★★★★★ | 拍照+相册分类/Notability |
| **笔记杂乱无章** | 照片、文档、手写散落各处，期末找不到对应内容 | 每周 | ★★★★☆ | 相册文件夹/百度网盘 |
| **DDL 总忘记** | 作业截止、考试时间、小组任务经常遗忘或拖延 | 每周 | ★★★★★ | 手机备忘录/日历 |
| **复习资料零散** | 学期末复习资料散落在各APP、群文件、相册，整合耗时 | 每学期 | ★★★★☆ | 自行整理/学长资料 |
| **网课容易错过** | 慕课/超星等网课截止时间多且分散，容易漏看漏交 | 每月 | ★★★★☆ | 课程群通知 |
| **学习缺乏动力** | 独自学习容易分心、懈怠，缺少正向反馈和鼓励 | 经常 | ★★★☆☆ | 番茄钟/自习室APP |
| **逃课后果不可知** | 偶尔翘课但不知道讲了什么重点、有没有点名签到 | 偶尔 | ★★★☆☆ | 问同学 |

### 明确不做（拒绝同质化）

| 冗余功能 | 原因 |
|---------|------|
| ❌ 食堂菜单/空位查询 | 与AI能力无关，低频使用 |
| ❌ 快递查询/取件码 | 已有菜鸟等成熟方案 |
| ❌ 天气查询 | 系统自带功能 |
| ❌ 教务系统对接/课表导入 | 学校系统封闭难对接，非AI强项 |
| ❌ 校园卡充值/消费查询 | 涉及资金安全，非快应用优势 |
| ❌ 二手交易/失物招领 | 非AI核心能力，运营成本高 |
| ❌ 社团活动发布 | 与学习提效核心定位不符 |

## 3.2 核心差异化

```
蓝心校园管家 vs 通用AI助手（豆包/文心一言）
  ├─ 场景垂直：专为课堂学习场景优化，而非通用对话
  ├─ 端侧能力：无网/弱网环境下仍可拍照识别笔记
  ├─ 闭环设计：记→提→复→盘，而非单次问答
  └─ 快应用免装：即点即用，不占存储

蓝心校园管家 vs 笔记类App（Notability/有道云）
  ├─ AI原生：拍照即结构化，而非纯手动整理
  ├─ 时间管理：笔记自动关联DDL提醒
  ├─ 主动服务：智能体自动整理、推送复习提醒
  └─ 轻量免装：快应用 vs 几百MB原生App

蓝心校园管家 vs 校园工具类（学习通/番茄Todo）
  ├─ 多模态：图文语音混合输入
  ├─ AI排版：公式、图表、代码自动识别
  ├─ 鼓励式互动：不是冷冰冰的工具，是学习搭子
  └─ 离线可用：端侧模型保障基础体验
```

| 冷启动优化 | 预加载首页数据、缓存上一个会话状态 | 冷启 < 1s |

## 12.4 评审可能提问 & 应对话术

| 可能问题 | 应对话术 |
|---------|---------|
| **和豆包/文心一言有什么区别？** | "我们是场景垂直的 AI 学习伴侣。通用 AI 助手解决的是"问什么答什么"，我们解决的是"课堂拍照→结构化笔记→DDL提醒→期末复习"的完整闭环。而且我们有端侧模型，教室没网也能用，这是通用 AI 做不到的。" |

| **为什么不做课表导入、教务对接？** | "我们坚持"极致聚焦"策略。课表解析我们用 AI 拍照识别（通用型），不需要对接学校系统。食堂、快递等功能已有成熟 App，我们做也无法比他们更好。我们的核心价值在笔记+时间管理这两个大学生最痛的场景，做深做透。" |
| **快应用能承载端侧模型吗？** | "3B 模型经过 INT8 量化后约 1.2GB，我们通过懒加载（使用时才加载）+ 分片下载 + 模型缓存策略，首包仅 800KB。推理使用 vivo 手机的 NPU 加速，首次加载约 3s，后续推理 < 500ms。同时我们做了兜底方案：模型下载中或加载失败时自动降级为纯云端模式。" |
| **商业化路径？** | "当前聚焦参赛和产品打磨。未来方向：① 基础功能免费（笔记+DDL）② AI 高级功能订阅制（思维导图 + 智能体周报）③ 与 vivo 生态合作（快应用预装/流量入口）。目标用户 6000 万在校大学生，按 5% 付费转化率、月费 ¥9.9 计算，潜在年收入约 3.5 亿。" |

# 十三、附录：技术选型详细说明

## 13.1 快应用 vs 原生 App 选型依据

| 维度 | 快应用 | 原生 App | 选型结论 |
|------|--------|---------|---------|
| 安装成本 | 免安装，即点即用 | 需下载安装（50-200MB） | ✅ 快应用胜 |
| 包体限制 | < 2MB（可扩展至 10MB） | 无严格限制 | 需要控制包体 |
| 端侧模型集成 | 通过 SDK 插件支持 | 原生集成 | 快应用可行 |
| 推送能力 | vivo 推送 | 各厂商推送 | 基本持平 |
| 开发效率 | 类 Vue 语法，快 | 原生开发慢 | ✅ 快应用胜 |
| 性能 | 接近原生 | 最优 | 基本持平 |
| 用户触达 | 负一屏/全局搜索入口 | 应用商店 | ✅ 快应用胜（vivo 生态） |

**结论**：基于 vivo 原生生态 + 竞赛场景，快应用是最优选择。端侧模型通过 vivo 提供的 SDK 插件机制接入。
